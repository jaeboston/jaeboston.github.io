    git BASIC commands



    To create a new repository for a new project
    git init <project-name>


    To create a new repository for an existing project
    cd <project-name>
git init


    To create a copy of an existing repository
    git clone
    1. touch .gitignore and choose extension not to check in
    2. git add
    3. git commit -m
    4. git push origin master
    5. git checkout filename ( before filename was added and bring in the existing copy)
    6. git rm (remove added files )



    Branching related commands



    To list existing branch:
    git branch
    To create a new branch:
    git branch <branchname>
    Go the the branch you want to work

    (How can I switch from one branch to another? )

    git checkout <branchname>

    git won’t let you do this unless all files are committed

    to create a new branch then checkout to the branch
    git checkout -b <branchname>

    the content of the new branch is same as the previous branch

    To compare branches
    git diff branch-1..branch-2


    Merging: check the branch which will stay (destination)
    git checkout <master>
git merge <source_branch> <destination_branch(master)>
    Delete a branch after merging
    git branch -d xxx
    Delete local branch (to force it use -D)
    git branch -d branch_name
    Resolve conflict on merging

    after merging error,

    git status

    it will display the file name with conflict Edit the file to resolve the conflict

    git add thefilename
git commit -m "message"
    Undo git add (files are already added)
    git reset HEAD *
    Renaming

    Assuming you’d like to change a folder’s name from oldfolder to newfolder

    git mv oldfolder newfolder

    Remove (delete ) a file which is already committed
    git rm filename

    it stages the file then needs to be comitted

    If there’s already a newfolder in your repo and you’d like to overwrite it use –force

    git mv -f oldfolder newfolder

    Do not forget: you have to add the changes to the index and commit them after renaming with git mv.

    git add -u newfolder
git commit -m "changed the foldername whaddup"
    set up remote repository
    git remote add origin remote_repo_url
    view existing remote repository
    git remote -v
    Cloneing existing reposiroty
    git clone https://github.com/coreui/coreui-free-bootstrap-admin-template.git
    configure –global (–system, –local) name
    git config --global user.name jae
git config --list --global
    delete un-staged changes (not added)
    git checkout -- filename
    delete staged changed (already added)
    git reset HEAD filename (will un-stage the file) / dirname (will un-state the files in the dir)
git checkout -- filename (will delete the un-staged file) / . (will delete all un-staged files in the current directory)
    To compare modified file with the previous version
    git diff filename
    To tag a version number
    git tag -a v1.2019.07.23 -m "tagging a version number"



    Configuration related



    EOL (end of line)

    Add EOL for windows not for linux 

    git config --global core.autocrlf true
